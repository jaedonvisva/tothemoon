01 — Concept
Design Direction
✈️
Tone
Bright sky arcade. Daytime cobalt gradients, cartoon cloud puffs, a green ground strip at the bottom — exactly like Aviamasters. Cheerful but tense. The plane is the hero of every screen state.
🎮
Feel
Gameplay-first. P&L numbers float up like RPG damage popups. Wins are big bold green bursts. Liquidations are an explosion + screen shake. Missiles fly past as ambient tension. Always in motion.
📐
Key Metaphor
Plane altitude = your P&L. Higher it flies, bigger you're winning. Plane banks with rate of change. Ground strip = liquidation zone. Getting close to ground = danger. Sky = possibility.

05 — Motion Design
Animations & Interactions
☁️ Cloud Scroll — always running, 2 layers
Front clouds at 12s, back clouds at 25s. Creates constant sense of flight even without an active trade. Pure CSS keyframes, no JS needed. 3–4 cloud shapes per layer, staggered with animation-delay.
@keyframes cloudScroll { from { transform:translateX(110%) } to { transform:translateX(-110%) } } .cloud-near { animation: cloudScroll 12s linear infinite; } .cloud-far { animation: cloudScroll 25s linear infinite; animation-delay: -8s; }
✈️ Plane Altitude — real-time P&L tracker
Map pnl_percent (clamped: -5% → +1000%) to plane Y position on canvas. Ground strip = death zone. Smooth lerp every 100ms poll. Plane rotates based on rate of change — banking up when climbing, nose-down when falling.
const yNorm = (pnl - minPnl) / (maxPnl - minPnl); // 0→1 targetY = canvasH * (1 - yNorm) + groundH; planeY = lerp(planeY, targetY, 0.12); // smooth follow planeRot = lerp(planeRot, deltaY * -0.3, 0.1); // banking
💫 Floating P&L Numbers — RPG damage popups
Every ~500ms, a "+X%" number spawns at the plane's current position and floats upward 50px while fading. Green for positive delta, red for negative. Font size scales with magnitude — a +200% tick should feel huge. Cap at 5 simultaneous DOM nodes.
@keyframes floatUp { 0% { transform:translateY(0); opacity:1; } 100% { transform:translateY(-50px); opacity:0; } } .pnl-float { animation: floatUp 1.2s ease-out forwards; }
💥 Liquidation Sequence — ~900ms total
1) Missile visually hits plane — explosion emoji spawns (frame 0). 2) Canvas shakes (CSS translate keyframe, 300ms). 3) Plane spirals down and off-screen (500ms ease-in). 4) Red flash overlay (200ms). 5) "Plane Down!" modal springs up from bottom (Framer Motion spring). Max drama, minimum delay.
🎉 Win Sequence — the money shot
1) Plane accelerates and flies off top of canvas (400ms). 2) Confetti burst (green/yellow/white). 3) Sky flushes bright — canvas bg shifts to lighter blue. 4) Cloud scroll speeds up 3×, returns to normal over 3s. 5) "To The Moon!" modal slides up with big Fredoka One number counting from 0 to final value.
🚀 Missiles — ambient danger cues
Missile emoji objects fly right-to-left at varying Y positions and speeds as background decoration. Baseline: 1 missile every 4s. When pnl approaches stop-loss threshold (<20% remaining margin): increase to 1 every 1.5s and one missile "locks on" and circles the plane to signal imminent liquidation.

⚛️
Stack
React 18 + TypeScript. Framer Motion for plane, modal, popup animations. Tailwind for layout. Zustand for position state. React Query for polling with adaptive intervals.
🎨
Sky Canvas
Pure CSS + absolute positioning — no canvas API needed. Clouds: CSS keyframe translateX. Plane: CSS transform with requestAnimationFrame lerp. Missiles: CSS keyframe from right edge. No WebGL required.
⚡
Performance
Pause all polling when tab is hidden (visibilityAPI). Only poll /positions/:id at 100ms while OPEN. Drop to 2s when PENDING. Cap floating P&L nodes at 5 with will-change: transform applied.
